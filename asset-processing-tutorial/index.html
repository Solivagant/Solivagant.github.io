Unity Automatic Asset Processing Tutorial

23 September 2015 on unity, tutorial

One of the most frequent questions I see online about Unity and pixel art is how to set up your sprites so that they look pixel perfect.

There's a couple of ways outlined out there and most likely all of them involve setting up your asset imports to specific settings.

So you go about your business, select all the sprites you have on the project, and set it up. But then a couple of weeks later, you've got some new art, you gotta go and do it all again.

And again!

Enter automatic asset processing scripts.

The Unity docs describe them as:

AssetPostprocessor lets you hook into the import pipeline and run scripts prior or after importing assets.

Here's a very simple example of an AssetPostprocessor.

1 - Create a subclass of AssetPostprocessor and save it in an Editor folder.
2 - Implement OnPreProcessTexture. You get access to the TextureImporter via the assetImporter field. It's in this importer that you change the settings.

    public void OnPreprocessTexture()
    {
        TextureType type = GetTextType(assetPath);
        if(type == TextureType.UnitySprite)
        {
            SetUnitySprite (assetImporter as TextureImporter);
        }
    }

    void SetUnitySprite(TextureImporter importer)
    {
        importer.mipmapEnabled = false;
        importer.maxTextureSize = 4096;
        importer.textureFormat = TextureImporterFormat.AutomaticTruecolor;
        importer.textureType = TextureImporterType.Sprite;
        importer.spritePixelsPerUnit = PIXELS_P_UNIT;
        importer.filterMode = UnityEngine.FilterMode.Point;
    }       
    TextureType GetTextType (string assetPath)
    {       if(assetPath.Contains("Resources/Sprites"))
        {
            return TextureType.UnitySprite;
        }
        else
        {
            return TextureType.Other;
        }
    }
}

This will work off the bat, just try adding any new images to the specified path and it'll be imported with the specified settings.

It's a huge time saver, and you can use it in all sorts of data, including XML, for example, so you could import weapon statistics by updating an XML file, rather than a static class somewhere or another clunky solution.
